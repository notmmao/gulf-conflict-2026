# Gulf Conflict 2026 项目使用说明

## 项目概述

Gulf Conflict 2026 是一个专门跟踪2026年美以伊战争/冲突信息的自动化项目。本项目使用 Qwen Code 自动跟踪战争进展，生成客观战报和概况文档，并通过 Markdown 转 HTML 的方式部署为静态网站。

## 项目结构

```
gulf-conflict-2026/
├── AI/                    # AI 任务相关文件
├── content/               # Markdown 内容源文件
│   ├── index.md          # 主页文档
│   ├── 美以伊战争 - 概况.md
│   ├── 美以伊战争 - 战报.md
│   ├── 战报更新.md
│   └── 更新记录.md
├── public/                # 生成的 HTML 文件
│   └── styles/           # CSS 样式文件
├── scripts/              # 处理脚本
│   └── md2html.py        # Markdown 转 HTML 脚本
├── templates/            # HTML 模板
│   └── default.html
├── build.sh              # 构建脚本
├── publish.sh            # 发布脚本
├── commit_push.sh        # 提交并推送脚本
└── update.sh             # 更新脚本
```

## 主要功能

### 1. 战争跟踪与报告

- **概况文档**：记录战争背景、参战方、力量对比、时间线等基本客观事实
- **每日战报**：搜索并更新自开战以来每日的详细战报，包括双方战况、消息出处以及评估置信度
- **更新记录**：记录每次更新的日期和主要内容

### 2. 自动化更新逻辑

项目采用智能更新机制：

1. **首次运行**（更新记录.md 不存在）：
   - 创建所有文档，更新战况至最新日期和时间

2. **后续运行**（更新记录.md 已存在）：
   - 读取更新记录最后一行，获取上次更新日期和时间
   - 如果当前日期时间 > 更新日期，则搜索并更新内容
   - 在更新记录.md 末尾追加新的更新记录

### 3. Markdown 转 HTML

- 支持动态内容注入（使用 `<!-- include: filename.md -->` 语法）
- 自动生成目录（TOC）
- 修复 HTML 中的 href 链接空格问题
- 采用微信公众号风格的界面设计
- 支持响应式布局

## 使用方法

### 1. 环境准备

确保您的系统已安装：
- Python 3.x
- Git（可选，用于版本控制）

安装必要的 Python 库：
```bash
pip install markdown
```

### 2. 构建网站

执行构建脚本，将 Markdown 文件转换为 HTML：

```bash
./build.sh
```

脚本会自动：
- 检查 Python 环境
- 安装必要的库
- 转换所有 Markdown 文件为 HTML
- 复制 CSS 样式文件

### 3. 发布网站

项目支持通过 Vercel 部署：

1. 安装 Vercel CLI：
```bash
npm install -g vercel
```

2. 部署到 Vercel：
```bash
./publish.sh
```

### 4. 手动更新内容

如果需要手动更新内容，可以编辑 `content/` 目录下的 Markdown 文件：

- `index.md`：主页，包含导航和更新记录
- `美以伊战争 - 概况.md`：战争概况信息
- `美以伊战争 - 战报.md`：每日战报详情
- `战报更新.md`：战报更新说明
- `更新记录.md`：历史更新记录

## 内容组织

### 主页 (index.md)

包含文档导航、更新记录和战报更新说明。使用 `<!-- include: filename.md -->` 语法动态引入其他文件。

### 战争概况

包含以下部分：
- 战争背景（伊核问题谈判破裂、"12 日战争"铺垫等）
- 参战方及战争目标
- 军事力量对比
- 战争时间线概览
- 伤亡与损失统计
- 地区外溢效应
- 国际反应
- 战争特点
- 停战条件

### 每日战报

按日期记录战争进展，包括：
- 双方战况
- 消息来源
- 评估置信度

## 特色功能

### 1. 智能内容注入

支持在 Markdown 文件中使用 `<!-- include: filename.md -->` 语法动态引入其他文件内容。

### 2. 自动生成目录

使用 `markdown` 库的 TOC 扩展自动生成文章目录，支持锚点跳转。

### 3. 响应式设计

采用微信公众号风格的设计，在各种设备上都能良好显示。

### 4. 返回顶部功能

自动添加返回顶部按钮，提升阅读体验。

## 自定义配置

### 修改样式

编辑 `public/styles/wechat.css` 文件可以自定义网站样式。

### 修改模板

编辑 `templates/default.html` 文件可以修改 HTML 模板结构。

### 调整构建参数

编辑 `scripts/md2html.py` 文件可以调整转换参数，如：
- 输入/输出目录
- 模板文件路径
- CSS 文件路径

## 故障排除

### 1. Python 环境问题

如果遇到 Python 相关错误：
- 确保已安装 Python 3.x
- 检查 `markdown` 库是否正确安装

### 2. 路径问题

如果遇到文件路径错误：
- 检查所有文件路径是否正确
- 确保所有必要文件存在

### 3. 构建失败

如果构建失败：
- 检查 Markdown 文件语法是否正确
- 查看 `build.sh` 的输出获取错误信息

## 版本控制

项目使用 Git 进行版本控制。可以通过 `commit_push.sh` 脚本提交更改并推送到远程仓库。

## 注意事项

1. 确保所有 Markdown 文件使用 UTF-8 编码
2. 图片文件应放置在适当的位置，并使用相对路径引用
3. 定期备份重要内容
4. 遵循 Markdown 语法规范，确保内容正确显示
